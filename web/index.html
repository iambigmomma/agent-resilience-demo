Summary

The only difference

Same agent.py, same fault, no if lane == anywhere:

SINGLE = Lane(candidates=(PRIMARY,))         # nowhere to go → dies
ROUTED = Lane(candidates=(PRIMARY, ALT))     # somewhere to go → finishes
notes & caveats Not a latency benchmark — the routed lane pays for an extra failed request before it fails over; the only measurement is whether the task finished. The fault is synthetic (local proxy, deterministic per run) and says nothing about any provider's real rate limits. Cost figures are rough estimates.

The only change

The client never changed. The entire migration is one config write:

PUT /v2/gen-ai/models/routers/{id}   models: ["openai-gpt-oss-20b", "llama3.3-70b-instruct"]
notes & caveats Observed propagation ~2s; rollback is the same write in reverse. The pinned lane would need an edit and a redeploy to do either. Latency figures are whatever happened during this run — not a benchmark. The ranking is restored after every run. This drives the real DigitalOcean Inference Router; nothing is simulated except in MOCK mode.
