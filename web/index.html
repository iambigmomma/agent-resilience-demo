The only difference

Both lanes ran the same agent.py against the same fault. There is no if lane == anywhere in client.py. The difference is the length of one tuple in lanes.py — the right lane had somewhere else to go.

# lanes.py SINGLE = Lane(candidates=(PRIMARY,)) # ← nowhere to go: burns its retries, dies ROUTED = Lane(candidates=(PRIMARY, ALT)) # ← routes around it, finishes the task

This is not a latency benchmark. The routed lane is often slower — it makes an extra failed request before failing over. The only thing being measured is whether the task finished. The fault is a synthetic 429 injected locally; it says nothing about any provider's real rate limits.
